Detailed Description

Implementation of the Evo class that acts on species optimum as well as interactions.

The EcoEvo class implements the getInds() method differently from StdEvo, since it creates I2 objects to which it returns a list of pointers. The runSelection() method remains the same, but getEvo() also updates species optimum.

Controller method for updating species interactions/optimum.

Calls getInds() to get a list of individuals on which to call runSelection(). Then calculates the mean of all interaction values, as well as optimum, from selected individuals, and sets this as the new interaction values, and optimum, for the species.

Parameters
speciesListList containing all (alive) species present in the environment (used to calculate survival/reproduction probability of indivduals).
specSpecies for which we will calculate evolutionary change (parameters are automatically updated during call).
resolutionHow many discrete individuals are contained in one unit of density.

Method for creating individuals from species.

This method divides a species with continous density into a discrete number of individuals numInds = round(species.density) * resolution + 1<> (+1 to avoid divide-by-zero errors). Interaction values for each individual are generated from a gaussian distribution where is species interaction value, and is evolution range for that interaction. Individual optimum is calculated in the same manner, with being species optimum, and optRange. Objects pointed to are of class I2 instead of Individual (take environmental constant into account).

Parameters
speciesListList containing all (alive) species present in the environment (used to calculate survival/reproduction probability of indivduals).
specSpecies from which we will generate individuals.
resolutionHow many discrete individuals are contained in one unit of density.
